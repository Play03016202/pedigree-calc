# 近親交配対応家系図アプリケーション — 開発フロー

## 概要

動物（人間含む）の近親交配に対応した家系図を視覚的に表示し、**近交係数（F値）を自動計算**するローカル動作の HTML シングルファイルアプリケーション。  
対象環境：Windows 10 / Chrome。外部サーバー不要、ファイルをブラウザで開くだけで動作する。

---

## 要件・仕様

### 必須（Phase 1）

- 個体の追加・編集・削除（名前、性別、父・母の指定）
- 世代別階層レイアウトによる家系図描画（上=古い世代）
- 表形式法（Tabular Method）による近交係数 F の自動計算
- ノードの色による F 値の視覚化（緑→黄→赤）と数値ラベル表示
- CSV インポート／エクスポート
- ズーム・パン操作

### 強く推奨（Phase 2）

- ノードホバーでツールチップ（F値・ECG・個体情報）
- ノードクリックで祖先・子孫のハイライト
- 交配シミュレーション（オス×メスを選択 → 産仔の予測 F 値をリアルタイム表示）
- 近交エッジの色・線種による視覚的強調（通常エッジ=グレー実線、近交エッジ=橙曲線）
- 世代フィルタースライダー
- 近交パス可視化（共通祖先とパスのハイライト）

### あれば良い（Phase 3）

- 集団統計ダッシュボード（平均F・Ne・分布ヒストグラム）
- 世代トレンドグラフ（ΔF・Ne）
- ECG（等価完全世代数）の計算・表示と精度警告
- 最影響祖先ランキング
- ノード展開モード（共通祖先の複製表示への切り替え）
- JSON 形式での保存・読み込み

## アーキテクチャ・設計

### 技術スタック

```
単一 HTML ファイル構成（外部 CDN 参照可）
├── レンダリング：SVG + D3.js v7
│   └── 理由：DOM 操作が容易でインタラクション実装コストが低い
│           100個体程度まで十分なパフォーマンス
├── レイアウト：Sugiyama フレームワーク（elk.js または自前実装）
│   └── 理由：世代ラインが保証される唯一の標準手法
├── 計算：純粋 JavaScript（表形式法 / NRM）
└── スタイル：CSS カスタムプロパティ（テーマ対応）
```

### データ構造

```javascript
// 個体エントリ
{
  id: "uuid-v4",
  name: "string",
  sex: "M" | "F" | "U",
  birthYear: number | null,
  sireId: "uuid" | null,   // 父
  damId: "uuid" | null,    // 母
  breed: "string",
  notes: "string",
  // --- 自動計算（変更禁止） ---
  F: number,               // 近交係数（0〜0.5）
  ECG: number,             // 等価完全世代数
  generationDepth: number  // レイアウト用世代番号
}

// NRM キャッシュ（相加的血縁行列）
Map<"id1:id2", number>    // "id1" < "id2" でキー正規化
```

### 近交係数計算アルゴリズム

**表形式法（Tabular Method / NRM法）** を採用。

- 計算量 O(n²)、最大 500 個体程度まで問題なし
- 全個体を世代番号順（親→子）にソートし NRM を構築
- `F(i) = 0.5 × NRM[sire][dam]`
- `NRM[i][i] = 1 + F(i)`
- `NRM[i][j] = 0.5 × (NRM[j][sire] + NRM[j][dam])`
- 交配シミュレーション：`F_child = 0.5 × NRM[候補オス][候補メス]`

> Wright パス係数法は小規模家系の教育的可視化用にサブ実装として保持（近交パス表示機能で使用）

### レイアウト設計方針

| 原則 | 実装 | 根拠 |
|------|------|------|
| 世代ラインの明確化 | 横方向の世代グリッド線（薄い点線）を表示 | Ballweg et al. (2017) — 階層の深さが知覚の最重要因子 |
| 同世代 Y 座標統一 | Sugiyama Phase 2 でレイヤー固定 | Huang et al. (2006) — 空間的整理が理解を左右 |
| エッジクロスより整列優先 | バリセンター法で近似 | 知覚研究でクロスは影響小 |
| 近交エッジの明示 | 橙・破線・ベジェ曲線 | 通常エッジと区別し構造を明示 |
| 初期表示は 3 世代 | 世代フィルタースライダーで制御 | Progressive Disclosure |

### F 値の色表示仕様

```
F < 0.0625 (6.25%)  → 緑系   #4CAF50  許容範囲
F = 0.0625〜0.125   → 黄系   #FFC107  注意（いとこ交配相当）
F = 0.125〜0.25     → 橙系   #FF9800  高リスク（半兄妹交配相当）
F > 0.25            → 赤系   #F44336  非常に高リスク
```
色弱対応：色相のみに依存せず、枠線の太さとラベル数値を必ず併用。

## 開発フロー

### Phase 1（コアエンジン実装）— 推定 3〜5 日

1. **データ管理モジュール**
   - 個体の CRUD 操作
   - UUID 生成、親子関係バリデーション（ループ・自己参照の防止）
   - 世代深度の計算（再帰 DFS）

2. **NRM 計算エンジン**
   - 世代順ソート → NRM 構築 → F 値の算出
   - キャッシュ管理（個体追加・編集時の差分再計算）
   - 単体テスト：既知の F 値（兄妹交配=0.25、いとこ=0.0625）で検証

3. **Sugiyama レイアウトエンジン**
   - 世代レイヤー割り当て
   - バリセンター法によるノード順序最適化
   - XY 座標の計算

4. **SVG レンダラー基礎**
   - ノード描画（矩形＋名前ラベル＋F 値バッジ）
   - エッジ描画（通常エッジ：グレー直線）
   - D3.js ズーム＆パン

5. **個体入力フォーム**
   - サイドパネルで追加・編集
   - 父・母セレクトボックス（自分より世代が上の個体のみ表示）

6. **CSV インポート/エクスポート**
   - カラム：`id,name,sex,birthYear,sireId,damId,breed,notes`
   - サンプルデータ付き

---

### Phase 2（インタラクションと育種機能）— 推定 3〜4 日

7. **ノードインタラクション**
   - ホバー：ツールチップ（名前・F値・ECG・父・母）
   - クリック：祖先・子孫の全エッジをハイライト（青・橙）
   - ダブルクリック：詳細パネルを開く

8. **近交エッジの可視化**
   - NRM 計算時に近交を形成するエッジを特定
   - 橙色ベジェ曲線で描画

9. **交配シミュレーターパネル**
   - オス・メスを選択（ドロップダウン or ノードクリック選択）
   - 産仔の予測 F 値をリアルタイム表示（0.5 × NRM[オス][メス]）
   - F 値の閾値判定とカラーバッジ表示

10. **近交パス可視化**
    - クリック個体の F 値 > 0 の場合、Wright パス係数法で共通祖先・経路を列挙
    - 該当ノード・エッジをアニメーションでハイライト

11. **世代フィルタースライダー**
    - 選択個体を起点に N 世代以内のみ表示

---

### Phase 3（分析ダッシュボード）— 推定 2〜3 日

12. **集団統計パネル**
    - 平均F・最大F・Ne（= 1 / 2ΔF）の表示
    - F 値分布ヒストグラム（SVG バーチャート）
    - Ne < 50 で警告バナー

13. **世代トレンドグラフ**
    - 世代ごとの平均 F の折れ線グラフ
    - ΔF の可視化

14. **ECG 精度警告**
    - 各個体の ECG を計算
    - ECG < 3 の場合「計算精度が低い可能性」をツールチップに表示

15. **最影響祖先ランキング**
    - 集団の遺伝的多様性への寄与率が高い上位 10 個体を表示

## 懸念点・リスク

| リスク | 対策 |
|--------|------|
| 近交が深い家系での NRM の計算量膨大（n² が問題になるのは 300 個体以上） | キャッシュ管理と差分再計算で対応。500 個体を上限として警告表示 |
| Sugiyama レイアウトの near-crossing が増えすぎると視認性低下 | 近交エッジのみ曲線化して区別。バリセンター法の反復回数を増やす設定 |
| 片親不明（未登録）個体でのF値精度低下 | ECG 計算で精度を定量化しツールチップに表示。「データ不完全」警告 |
| 循環参照（A の祖先に A 自身が現れる）防止 | 親指定時のリアルタイムバリデーション（DFS で循環チェック） |
| ブラウザのローカルストレージ制限（5MB） | JSON エクスポートを推奨。LocalStorage はキャッシュのみ使用 |
